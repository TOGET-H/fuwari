# 智能旅游助手 (Node Travel Agent) — 项目详解文档

  
地址：https://github.com/TOGET-H/node_travel_agent
---

  

## 一、项目概述

  

本项目是一个**全栈 AI 智能旅游规划助手**，由前端 Vue 3 SPA 和后端 Express.js API 服务组成。核心功能是：

- 用户输入**目的地、预算、天数**，由 AI（大语言模型）生成完整旅行计划

- 用户可与 AI **实时流式对话**，获取旅行建议

- 后端调用 DeepSeek API（兼容 OpenAI 格式），通过 LangChain 封装

  

### 技术栈

  

| 层级     | 技术                                      |

| -------- | ----------------------------------------- |

| 前端框架 | Vue 3 (Composition API + `<script setup>`) |

| 构建工具 | Vite 8                                    |

| UI 组件库 | Vant 4 (移动端组件库)                     |

| 路由     | Vue Router 4 (Hash History)                |

| HTTP    | Axios（普通请求）+ Fetch API（SSE 流式）     |

| 后端     | Express.js 5                              |

| AI SDK  | LangChain (`@langchain/openai`)           |

| AI 模型  | DeepSeek (deepseek-chat / DeepSeek-V4)    |

| 运行时   | Node.js                                   |

### 3.6 HTTP 工具模块：`node_travel_agent/src/utils/request.js`

  

#### 3.6.1 Axios 实例（普通 JSON 请求）

  

```js

const request = axios.create({

    baseURL: "/api",        // 所有请求以 /api 为前缀

    timeout: 60000,         // 60 秒超时

})

```

  

- **请求拦截器：** 透传 config（预留了添加 Token 等扩展点）

- **响应拦截器：** 自动解包 `response.data`（调用者拿到的直接是业务数据），错误时 `console.error` 并继续 reject

- **导出函数：** `post(url, data)` → `request.post(url, data)` / `get(url, params)` → `request.get(url, { params })`

  

#### 3.6.2 SSE 流式请求（`stream` 函数）

  

**签名：**

```js

stream(url, data, onChunk, onComplete, onError, signal) → cancelFunction

```

  

**工作流程：**

  

```

1. fetch('/api' + url, { method: 'POST', body: JSON })

       │

2. response.body.getReader() 获取 ReadableStream reader

       │

3. 循环 reader.read()，用 TextDecoder 增量解码

       │

4. 积累到 buffer 中，按 '\n\n' 分割出 SSE 事件块

       │

5. 解析每个事件块：

   ├── "event: error"   → 抛出异常，触发 onError

   ├── "event: end"     → 调用 onComplete，结束

   │   或 data.done === true

   └── "data: {...}"    → JSON.parse → 调用 onChunk(data)

       │

6. 返回 cancel 函数（设置 canceled=true + reader.cancel()）

```

  

**SSE 协议解析细节：**

- `event:` 行 → 提取事件类型（`error` / `end` / 空）

- `data:` 行 → JSON.parse 后作为回调参数

- 支持 `AbortController.signal` 外部取消

- 缓冲区残留处理：done 时若 buffer 还有未处理数据，调用 `parseSSELine` 尝试解析

### 3.8 AI 对话页：`node_travel_agent/src/views/Chat.vue`

  

**页面定位：** 与 AI 进行实时流式对话，获取旅行建议。

  

#### 3.8.1 核心机制：打字机效果 + SSE 流式

  

```

用户输入 "推荐三亚景点"

  │

  ├→ messages.push({ role:'user', content:'...' })

  ├→ messages.push({ role:'assistant', content:'' })    ← 创建空 AI 消息槽

  ├→ startTypewriter(aiMsg)                              ← 启动 20ms 间隔定时器

  └→ stream('/chat', { message }, onChunk, onComplete, onError)

        │

        ├── onChunk: feedChars(data.content)  → 逐字推入 typeQueue

        │      └── 定时器每 20ms 从 typeQueue 取一个字符追加到 aiMsg.content

        │

        ├── onComplete: stopTypewriter()  → 清空剩余队列，设置 sending=false

        └── onError: 追加错误信息到 aiMsg.content

```

  

#### 3.8.2 数据结构

  

| 变量           | 类型            | 说明                              |

| -------------- | --------------- | --------------------------------- |

| `messages`     | `ref(array)`    | 消息列表 `[{role, content}]`      |

| `inputText`    | `ref(string)`   | 输入框绑定值                       |

| `sending`      | `ref(boolean)`  | 是否正在发送/接收中                  |

| `chatBody`     | `ref(DOM)`      | 消息列表容器 DOM 引用               |

| `typeQueue`    | 模块级数组       | 打字机字符队列                      |

| `typeTimer`    | 模块级 ID        | 打字机 setInterval 句柄             |

| `streamCancel` | 模块级函数       | SSE 取消函数                       |

  

#### 3.8.3 关键函数

  

| 函数               | 功能                                                                 |

| ------------------ | -------------------------------------------------------------------- |

| `onSend()`         | 发送消息：校验→添加用户消息→创建空 AI 消息→启动 typing→调用 `stream()` |

| `onStop()`         | 停止生成：调用 `streamCancel()` 取消 SSE 流                            |

| `startTypewriter`  | 清空队列，启动 20ms 定时器逐字输出                                     |

| `feedChars(text)`  | 将字符串拆成字符数组推入 `typeQueue`                                   |

| `stopTypewriter`   | 停止定时器，将剩余字符一次性 flush，更新滚动位置                        |

| `scrollToBottom`   | 将 `chatBody` 滚动到最底部                                             |

  

#### 3.8.4 生命周期

  

| 钩子                | 行为                                                                    |

| ------------------- | ----------------------------------------------------------------------- |

| `onMounted`         | 添加 AI 欢迎消息（"你好！我是你的智能旅行助手 🌍..."）                    |

| `onBeforeUnmount`   | 清除 `typeTimer`、调用 `onStop()` 取消未完成的流请求                      |

  ├→ 从 route.query 读取 destination、budget、days、interests

  └→ fetchTripData()

        │

### 4.3 业务服务层：`server/src/services/travelService.js`

  

这是整个项目的**核心业务逻辑**所在，采用单例模式导出。

  

#### 4.3.1 构造函数 & LLM 初始化

  

```js

class TravelService {

  constructor() { this.initLLM() }

  

  initLLM() {

    const apiKey = process.env.API_KEY

    const baseURL = process.env.BASE_URL       // https://api.deepseek.com/v1

    const model = process.env.MODEL            // deepseek-chat

  

    this.llm = new ChatOpenAI({

      openAIApiKey: apiKey,

      configuration: { baseURL },

      modelName: model,

      temperature: 0.7,

    })

  }

}

```

  

- 使用 **LangChain** 的 `ChatOpenAI` 类，通过 `configuration.baseURL` 指向 DeepSeek 的 API 端点

- DeepSeek API 兼容 OpenAI SDK 格式，因此可以直接使用 LangChain 的 OpenAI 封装

- `temperature: 0.7` 保证 AI 输出既有创意又不至于过于随机

  

#### 4.3.2 `generatePlan()` — 生成完整旅行规划

  

**Prompt 结构：**

  

| 角色   | 内容                                                                     |

| ------ | ------------------------------------------------------------------------ |

| System | "你是一位资深的旅行规划师，拥有全球旅行经验..."（4 条专业要求）              |

| User   | 包含【目的地】【预算】【天数】的具体信息 + 四大输出结构要求（概览/行程/预算/贴士） |

  

**User Prompt 要求 AI 按四段输出：**

1. **一、行程概览** — 最佳出行季节、住宿区域、预算范围

2. **二、每日详细行程** — 每天按上午/中午/下午/晚上安排，标注活动/时间/费用

3. **三、预算明细表** — Markdown 表格格式（交通/住宿/餐饮/门票/其他/合计）

4. **四、实用贴士** — 交通建议 / 美食推荐 / 注意事项 / 省钱技巧

  

**返回值：**

```js

{

  raw: '一、行程概览\n...',        // AI 原始 Markdown 文本

  parsed: { overview, schedule, budget, tips, input }  // 结构化解析结果

}

```

  

#### 4.3.3 `chatStream()` — 流式对话

  

**流程：**

```

1. 构造 System Prompt（旅行规划师角色设定）

2. 调用 this.llm.stream([...]) 获得异步迭代器

3. for await (const chunk of chunks):

     stream.send({ content: chunk.content })   ← 逐块推送到 SSE

4. stream.end()

5. 异常时 stream.error(err)

```

  

#### 4.3.4 Markdown 解析系统（`parsePlan` 及子方法）

  

这是整个后端最精妙的部分——用**正则表达式**从 AI 返回的不规范 Markdown 中提取结构化数据。

  

**核心方法 `_section(text, marker)`：**

```js

// 截取两个中文数字标题之间的内容

// 例如：_section(text, '一、行程概览')

// 匹配从 "一、行程概览" 开始，到 "二、" 或 "三、" 等下一个标题前的内容

const re = new RegExp(`${marker}[\\s\\S]*?(?=\\n(?:#{1,4}\\s*)?[一二三四五六七八九十]、|$)`, 'i')

```

  

**各解析方法：**

  

| 方法               | 解析目标     | 技术策略                                                                            |

| ------------------ | ------------ | ----------------------------------------------------------------------------------- |

| `parseOverview()`  | 行程概览     | 正则 `/[-*]\s+\*{1,2}([^*]+?)\*{1,2}[：:]\s*(.+?)/g` 匹配 `- **键**：值` 格式        |

| `parseSchedule()`  | 每日行程     | 正则匹配 `第N天` / `Day N` 标题分割 → 调用 `_parseSlots()` 解析各时段                  |

| `_parseSlots()`    | 时段详情     | **三层降级策略：** ①表格格式 `\| 时段 \| 活动 \| 时间 \| 费用 \|` → ②列表格式 `* **时段：**` 带子项 → ③简单 `* **时段：** 文本` |

| `parseBudget()`    | 预算明细表   | 正则 `/^\|\s*([^\|]+?)\s*\|\s*([^\|]+?)\s*\|/gm` 提取 Markdown 表格的两列              |

| `parseTips()`      | 实用贴士     | 先按二级标题/粗体分割出各子类别 → 再用 `/^\s*(\d{1,2})[\.\)、]\s*([^\n]+)/g` 提取编号列表 |

  

**解析容错设计：**

- 同时支持 `### 一、` 和 `一、` 两种标题格式

- 支持 `- **键**：` / `* **键**：` / `- **键**:` 三种写法

- 预算表格自动跳过表头行（`项目`、`费用`）和分隔行（`---`）

- 行程解析有多层降级策略（表格 → 列表 → 简单行），最大程度兼容 AI 输出的多样性

### 4.4 SSE 流工具：`server/src/utils/streamUtils.js`

  

**导出函数：** `createStreamResponse(res)` → `{ send, end, error }`

  

#### 功能说明

  

1. **设置 SSE 响应头：**

   ```

   Content-Type: text/event-stream

   Cache-Control: no-cache

   Connection: keep-alive

   ```

  

2. **`send(data)`：** 写入 `data: {JSON}\n\n`（标准 SSE 数据块）

  

3. **`end()`：** 写入 `event: end\ndata: {"done":true}\n\n` 后关闭连接

  

4. **`error(err)`：** 写入 `event: error\ndata: {"error":"..."}\n\n` 后关闭连接

  

5. **断连处理：** `res.on('close', () => res.end())` 监听浏览器主动断开

### 两种通信模式对比

  

| 特性        | `/api/plan`（普通请求）          | `/api/chat`（SSE 流式）              |

| ----------- | ------------------------------- | ------------------------------------ |

| 前端实现    | `axios.post()` → `response.data` | `fetch()` + `ReadableStream` + 手动解析 |

| 后端实现    | `res.json()` 一次性返回           | `res.write()` 逐块推送                |

| 响应格式    | JSON                            | `text/event-stream`                  |

| 用户体验    | 加载中 → 一次显示                 | 逐字打字机效果，实时呈现                |

| 适用场景    | 完整规划生成（耗时较长）           | 对话交互（需要即时反馈）               |

## 七、设计亮点与特色

  

### 前端

  

| 亮点               | 说明                                                                     |

| ------------------ | ------------------------------------------------------------------------ |

| **打字机效果**      | Chat 和 Details 页面都实现了逐字显示效果的打字机，使用 `setInterval` + 字符队列实现 |

| **SSE 流式通信**    | 手动实现 SSE 协议解析（支持 `event:`、`data:` 行），兼容 AbortController 取消 |

| **暗色模式**        | 全局 CSS 变量 + `prefers-color-scheme: dark` 媒体查询，自动跟随系统主题     |

| **组件自动导入**    | `unplugin-vue-components` + Vant Resolver，无需手动 import，按需打包       |

| **渐变设计语言**    | 多处使用 `linear-gradient` 紫色/绿色/橙红渐变，统一的视觉风格               |

| **移动端适配**      | `viewport` 响应式 + `safe-area-inset-bottom` + `env()` 安全区适配          |

  

### 后端

  

| 亮点                | 说明                                                                     |

| ------------------- | ------------------------------------------------------------------------ |

| **三层降级解析**     | Markdown 解析对 AI 输出采用"表格 → 列表 → 简单行"三级降级策略，容错性强    |

| **正则结构化提取**   | 用纯正则从自由文本中提取 structured data，无需额外 AI 调用                  |

| **OpenAI 兼容层**    | 通过 LangChain `ChatOpenAI` + `baseURL` 配置接入 DeepSeek，代码与模型解耦  |

| **SSE 流封装**       | `createStreamResponse` 工厂函数，提供 `send/end/error` 统一接口            |

| **单例服务模式**     | `TravelService` 实例化一次，LLM 连接复用                                   |
