做项目时最常见的问题之一就是：前端页面写好了，后端接口也写好了，但联调时就是拿不到预期结果。相比直接猜问题在哪里，更可靠的方式是按请求链路一步一步排查。

## 先确认请求是否发出

第一步看浏览器 Network：

- 请求 URL 是否正确
- 请求方法是 `GET` 还是 `POST`
- 请求体是否真的带上了参数
- 状态码是 `200`、`400`、`401` 还是 `500`

很多问题并不是业务逻辑错了，而是请求根本没有打到正确接口。

## 再确认后端是否收到

如果前端请求看起来正常，就去后端加日志。重点看：

- `req.body` 是否有值
- 参数名是否和前端一致
- 是否经过了中间件解析
- 是否被鉴权或校验提前拦截

比如 Express 项目里，如果忘了 `express.json()`，后端就可能拿不到 JSON 请求体。

## 最后确认响应结构

前端经常假设接口返回：

```js
res.data.data
```

但实际后端可能返回：

```js
{
  code: 0,
  result: {}
}
```

这时接口本身没有失败，但前端读取字段错了，页面就会表现为空数据。

## 总结

接口联调不要靠感觉排查。我的习惯是按“请求发出 -> 后端接收 -> 业务处理 -> 响应结构 -> 页面渲染”这条链路逐步缩小范围。这个过程虽然慢一点，但能减少很多无效猜测。
