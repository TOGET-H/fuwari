在 AI 应用里，流式响应是一个很重要的体验优化。用户提问后，如果等模型完整生成再返回，页面会长时间没有反馈；而流式响应可以让内容一边生成一边展示，用户会明显感觉系统更快。

## 为什么需要流式响应

大模型生成内容通常不是瞬间完成的。一次旅行规划可能包含目的地分析、路线安排、餐饮建议、预算拆分等内容。如果全部生成完再返回，等待时间会比较长。

流式响应的价值在于：

- 降低用户等待焦虑
- 让页面更像真实对话
- 支持中途展示阶段性结果
- 方便后续做停止生成、重新生成等交互

## 前端如何处理

前端常见处理方式是使用 `fetch` 读取 `ReadableStream`，或者使用 SSE。核心思路都是：每收到一段文本，就追加到当前消息里。

伪代码大概是：

```js
const reader = response.body.getReader()
while (true) {
  const { done, value } = await reader.read()
  if (done) break
  appendMessage(decode(value))
}
```

这里真正需要注意的是状态管理。流式内容不是一次性赋值，而是不断追加，所以要保证 UI 更新不会阻塞，也要处理异常中断。

## 项目中的体验细节

在旅行助手项目里，流式响应适合配合这些细节：

- 生成中显示 loading 或光标
- 新内容自动滚动到底部
- 用户可以看到 AI 正在“思考”
- 错误时保留已生成内容，而不是整段清空

## 总结

AI 应用的难点不只是调用模型 API，更在于把模型输出变成稳定的产品体验。流式响应就是其中一个关键环节，它让系统从“等待结果”变成“参与生成过程”。
