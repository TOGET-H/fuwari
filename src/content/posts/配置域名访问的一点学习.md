在此之前计网相关知识虽然背了很多，但是实际运用的却很少，那么就从申请域名到配置 DNS 总体复盘一下。

## 申请域名

首先找一个服务商，腾讯云或者阿里云都可以，我这里选择了阿里云。


查询自己想要的域名然后购买。

## DNS 托管商

接下来复盘一下从阿里云申请域名后，修改 DNS 托管商的操作。

| 角色 | 主要职责 | 案例中对应的服务商 |
|---|---|---|
| **域名注册商** | 向 ICANN（互联网名称与数字地址分配机构）申请并注册域名，证明你是这个域名的合法所有者 | 阿里云 |
| **DNS 托管商** | 管理域名的 DNS 记录（A 记录、CNAME 记录等），告诉全世界"这个域名应该指向哪个 IP 地址的服务器" | 原本是阿里云，后来改成了 Cloudflare |

### 默认 DNS 托管

在阿里云付款购买 `example.com` 后，阿里云默认会为你提供免费的 DNS 托管服务，自动将域名的 **NS 记录（域名服务器记录）** 设置为阿里云自己的 DNS 服务器。此时，全世界所有 DNS 查询都会去阿里云的服务器询问 `example.com` 对应的 IP 地址。

### 迁移到 Cloudflare

在 Cloudflare 控制台输入 `example.com` 并点击"添加站点"后：

1. Cloudflare 会自动扫描你当前在阿里云 DNS 上的所有记录（A 记录、CNAME 记录、MX 记录等），并导入到自己的系统中
2. Cloudflare 会为你分配两个专属的 DNS 服务器地址

## DNS 查询过程

当 DNS 传播完成后，任何用户在浏览器输入 `example.com` 时，都会经历以下查询过程：

1. 用户的电脑先询问本地 DNS 服务器（比如电信、联通的 DNS，或者 `8.8.8.8`）："`example.com` 的 IP 地址是什么？"
2. 本地 DNS 服务器不知道，于是去问根域名服务器：".com 的 DNS 服务器在哪里？"
3. 根域名服务器返回 `.com` 顶级域名服务器的地址
4. 本地 DNS 服务器去问 `.com` 顶级域名服务器："`example.com` 的 DNS 服务器在哪里？"
5. `.com` 顶级域名服务器返回："去问 `erin.ns.cloudflare.com` 和 `roan.ns.cloudflare.com`"
6. 本地 DNS 服务器去问 Cloudflare 的 DNS 服务器："`example.com` 的 IP 地址是什么？"
7. Cloudflare 返回它配置好的 IP 地址（如果开启了 CDN，会返回 Cloudflare 的边缘节点 IP，而不是你的源服务器 IP）
8. 用户的电脑拿到 IP 地址后，向该 IP 发送请求，成功访问网站

## 为什么要用 Cloudflare？

- **免费 CDN 加速**：Cloudflare 在全球有数百个边缘节点，会缓存你的网站静态资源，让用户从离自己最近的节点访问，大幅提升访问速度
- **免费 DDoS 防护**：Cloudflare 可以抵御大规模的 DDoS 攻击，保护你的源服务器不被攻击打垮
- **免费 SSL 证书**：一键申请和续期 SSL 证书，让你的网站支持 HTTPS 访问
- **智能 DNS 解析**：根据用户的地理位置自动返回最优的节点 IP
- **防火墙和安全规则**：可以拦截恶意请求、爬虫和 SQL 注入等攻击
